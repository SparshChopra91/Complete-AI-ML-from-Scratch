An enterprise-grade, resource-constrained healthcare informatics platform designed to bridge the gap between machine learning accuracy and hospital economics. Built upon the UCI Cleveland Heart Disease dataset, this architecture utilizes a novel **Two-Tier Cascade Routing Engine**, a strict data-leakage-proof preprocessing pipeline, and a Generative AI layer to translate mathematical feature attribution into natural language clinical notes.

---

## 📌 The Academic & Clinical Novelty (The Gap)

Traditional machine learning research often assumes all patient diagnostics are available simultaneously at zero cost. This "all-at-once" approach is economically unfeasible for real-world clinics. Furthermore, standard Explainable AI (XAI) outputs abstract mathematical charts (like SHAP beeswarm plots) that place an unacceptable cognitive burden on fast-paced emergency physicians. 

**This architecture solves both flaws by introducing:**
1. **Financial Asymmetry:** A dynamic routing gate that attempts to discharge or admit patients using zero-cost vitals (Tier 1) before authorizing expensive, invasive metabolic lab work (Tier 2).
2. **Cognitive Load Reduction:** An automated pipeline that extracts local SHAP feature weights and routes them through the Google Gemini LLM to generate instant, plain-English medical summaries.

---

## 📐 System Architecture & Data Flow

The project is built on a strictly unified data preprocessing pipeline to prevent index shuffling, followed by a programmatic feature slice to feed two distinct algorithmic brains.

### 1. Unified Preprocessing Pipeline (`Data_Processing_final.py`)
To prevent data leakage, the entire dataset is ingested and split (`test_size=0.2, stratify=y`) before any mathematical transformations occur.
* **Continuous Imputation:** `IterativeImputer` (MICE) is strictly applied only to continuous numeric features (`age`, `trestbps`, `chol`, `thalch`).
* **Categorical Imputation:** Missing categorical data (like absent `restecg` results) are explicitly labeled as `Unknown` to allow the algorithm to learn the clinical significance of an un-ordered medical test.
* **Expansion & Scaling:** Features are expanded via `OneHotEncoder` and normalized boundingly into a $[0, 1]$ matrix via `MinMaxScaler`.

### 2. The Multi-Tier Matrix Slice
Once the dataset is unified and perfectly clean, the feature space is sliced into two asymmetric matrices:
* **Tier 1 (The Gatekeeper Matrix):** Contains only free, non-invasive intake metrics (`age`, `gender`, `cp`, `trestbps`).
* **Tier 2 (The Tie-Breaker Matrix):** Contains the complete diagnostic profile, introducing metabolic lab work (`chol`, `fbs`, `restecg`, `thalch`).

### 3. The Uncertainty Gate (`triage_engine.py`)
The system routes patients based on a calculated posterior probability $P(\text{Sick})$ generated by the Tier 1 Random Forest model:
* **Safe Discharge (Low Risk):** If $P \le 0.30$.
* **Immediate Escalation (High Risk):** If $P \ge 0.70$.
* **The Gray Zone (Trigger Labs):** If $0.30 < P < 0.70$. The system halts, requests the Tier 2 lab metrics, and passes the concatenated patient profile to the Tier 2 model for a definitive diagnosis.
